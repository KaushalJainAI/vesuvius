# Model Improvement Plan — SegmentInkNet

## Baseline Summary

- **Model**: SegmentInkNet — 3D encoder (Conv3d) → 2D decoder (Conv2d), ~0.24 M params
- **Input**: 33-layer Z-stack patches, 256×256
- **Loss**: 0.5 × BCE + 0.5 × soft-Dice, ignore band [0.4, 0.6]
- **Metric**: F0.5 @ threshold 0.4
- **Training data**: **All 11 labelled segments** (~72 GB, ~540 MB labels) ✅ confirmed
- **Val segment**: `20231221180251` (smallest)
- **Known weaknesses**: no skip connections, Z averaged away, weak augmentation, tiny capacity, noisy labels

---

## Critical Context — The GP Winner Used TimeSformer

The pseudo-labels we train on were generated by the **Grand Prize winning model (Nader / Farritor / Schilliger, 2023) — a TimeSformer-based architecture**. This has two direct implications:

1. **TimeSformer is proven for this task.** It is not a speculative experiment. It is the architecture that first successfully read Herculaneum scroll text from CT data. Moving it from "maybe" to a first-class candidate is correct.

2. **Our pseudo-label ceiling is TimeSformer's performance.** Training on its outputs, a CNN can at best reproduce what TimeSformer predicted. To genuinely surpass the pseudo-labels — and potentially read ink the GP winner missed — we need a model that reasons differently or better, ideally also using divided space-time attention.

This means the path to maximum F0.5 runs through TimeSformer, not away from it.

---

## What Changes With 11 Segments

With 10 training segments instead of 3:

| Assumption | With 3 segments | With 11 segments |
|---|---|---|
| Overfitting risk | High — model memorises 3 textures | Low — 10 distinct papyrus conditions |
| Model capacity ceiling | ~0.9 M (limited by data) | 2–5 M params is now reasonable |
| Augmentation role | Primary diversity source | Complementary — real diversity already in data |
| Transformer bottleneck risk | High overfitting risk | Low — enough variety to learn from |
| Pseudo-label refinement value | Moderate (3 segments) | High — covers full scroll |
| EfficientNet backbone | Risky (too few samples) | Viable |
| Cross-segment validation | Not feasible | Possible — leave-one-out or 5-fold |
| patches/epoch | 3 × 800 = 2400 | 10 × 400 = 4000 |

## Improvement Roadmap

---

### Tier 1 — High impact, low effort

#### 1.1 Add U-Net skip connections

**Problem**: The decoder receives only the bottleneck tensor (64 ch). All spatial detail — letter stroke edges, thin ink lines — is discarded in the encoder.

**Fix**: Save encoder feature maps before each pool, concatenate back in the decoder.

```
enc1_feat → ─────────────────────────────────────────┐
enc2_feat → ────────────────────────────┐             │
                                         ↓             ↓
bottleneck → zpool → dec1(cat enc2) → dec2(cat enc1) → head
```

**With 11 segments**: Skip connections receive richer, more diverse features — the model learns generalisable stroke edge representations rather than memorising 3 textures.

**Expected gain**: +3–6% F0.5.

---

#### 1.2 Increase model capacity

**Problem**: Channel widths (16→32→64) underfit 10 diverse segment textures.

**With 11 segments**: Enough data to push to a 4-stage encoder without overfitting.

| Stage | Baseline | V2 (11 segs) |
|---|---|---|
| enc1 | 1→16 | 1→32 |
| enc2 | 16→32 | 32→64 |
| enc3 | 32→64 | 64→128 |
| enc4 (new) | — | 128→256 |
| dec1 | 64→32 | 256→128 |
| dec2 | 32→16 | 128→64 |
| dec3 (new) | — | 64→32 |
| head | 16→1 | 32→1 |

New param count: ~3.5 M — comfortable in T4/V100 VRAM at batch_size=4.

**Expected gain**: +3–5% F0.5.

---

#### 1.3 Stronger augmentations

**With 11 segments**: Augmentation is no longer the primary diversity source. Focus shifts to simulating degradation modes not covered by the 11 segments.

**Add:**
- Gaussian noise: `vol + N(0, 0.03)` — CT scanner noise
- Brightness/contrast jitter: `vol × U(0.85, 1.15)` — inter-segment scan variation
- Random Z-layer dropout: zero out 1–3 slices — robustness to missing layers
- Elastic deformation (`alpha=50, sigma=5`) — local papyrus warping
- Random Z-shift: ±2 layers — surface detection variation

**Expected gain**: +1–3% F0.5.

---

#### 1.4 ~~Use all 11 segments~~ ✅ Confirmed

All 11 labelled segments used for local training. Val = `20231221180251`. `patches_per_seg = 400` → 4000 patches/epoch from 10 train segments.

### Tier 2 — Medium impact, moderate effort

#### 2.1 Replace Z average-pool with learned Z attention

**Problem**: `AdaptiveAvgPool3d` treats all Z layers equally. Ink depth varies across segments.

**With 11 segments**: The attention weights can learn a genuine prior over which Z ranges carry ink — not just one segment's profile.

```python
class ZAttentionPool(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.attn = nn.Conv3d(channels, channels, kernel_size=(1,1,1))

    def forward(self, x):               # (B, C, Z, H, W)
        w = torch.softmax(self.attn(x), dim=2)
        return (x * w).sum(dim=2)       # (B, C, H, W)
```

**Expected gain**: +2–3% F0.5.

---

#### 2.2 Focal loss component

**Problem**: Ink pixels are a minority (~5–15%). BCE over-weights easy background pixels.

Ink/background ratio varies across segments — focal loss adapts dynamically per pixel.

```
FL(p, y) = -α·(1-p)^γ·log(p)   for y=1
           -(1-α)·p^γ·log(1-p)  for y=0
```

Suggested: `γ=2, α=0.75`. Combined: `0.5 × FocalLoss + 0.5 × SoftDice`.

**Expected gain**: +1–3% F0.5.

---

#### 2.3 Cross-segment validation

**Problem**: Single fixed val segment may not represent performance across the full scroll.

**Fix**: Rotate val segment across 3–5 runs. Average F0.5 across runs as the true metric. Only possible with 11 segments.

**Expected gain**: Not a direct F0.5 gain — but a much more honest evaluation.

---

#### 2.4 Test-time augmentation (TTA)

Run inference under 8 orientations (4 rotations × 2 flips), average sigmoid outputs.

```python
def predict_tta(model, patch):
    preds = []
    for k in range(4):
        p = np.rot90(patch, k, axes=(1, 2))
        for flip in [False, True]:
            q = p[:, :, ::-1] if flip else p
            out = torch.sigmoid(model(to_tensor(q)))
            out = undo_transform(out, k, flip)
            preds.append(out)
    return np.mean(preds, axis=0)
```

**Expected gain**: +1–3% F0.5.

---

#### 2.5 Cosine warm-up + longer schedule

Each epoch is now ~4000 patches (10 segs × 400). Convergence is slower — a longer schedule with warm-up matters more.

**Fix**: 5-epoch linear warm-up + CosineAnnealingLR over 80 epochs.

**Expected gain**: +1–2% F0.5.

---

### Tier 3 — High impact, high effort

#### 3.1 Iterative pseudo-label refinement

**Problem**: Labels from one GP-winner run. The ignore band [0.4, 0.6] discards ~30% of pixels.

**With 11 segments**: Refinement covers the full scroll. Blended labels are more consistent because the model has seen diverse conditions across all 11.

**Plan**:
1. Train V2 CNN on all 11 segments with current pseudo-labels
2. Run inference across all 11 segments
3. Blend: `new_label = 0.6 × orig + 0.4 × pred`
4. Retrain on blended labels, repeat with `0.5/0.5` blend
5. Evaluate whether ignore band can narrow to [0.45, 0.55]

**Expected gain**: +4–8% F0.5.

---

#### 3.2 EfficientNet backbone (2.5D approach)

**With 11 segments**: Enough data to fine-tune a pretrained encoder without overfitting (was risky with 3 segments).

**Plan**: Collapse Z into projections → pretrained EfficientNet-B2 → FPN decoder.

```
vol (B, 33, H, W)
  ├─ mean over Z   → (B, 1, H, W) ─┐
  ├─ max  over Z   → (B, 1, H, W) ─┼→ (B, 3, H, W) → EfficientNet-B2 → FPN → head
  └─ std  over Z   → (B, 1, H, W) ─┘
```

**Downside**: Z collapsed to 3 statistics — loses explicit layer-by-layer depth reasoning.

**Expected gain**: +5–10% F0.5.

---

#### 3.3 Self-supervised Z-stack pre-training (MAE)

**With 11 segments**: ~72 GB of CT Z-stacks is a substantial corpus for self-supervised pre-training.

**Plan**: Mask 75% of Z-stack patches, train encoder to reconstruct them. Fine-tune on supervised ink detection.

**Expected gain**: +3–8% F0.5, especially for faint strokes under-represented in pseudo-labels.

---

### Tier 4 — Transformer Approaches

> TimeSformer is not speculative — it is the architecture the Grand Prize winning team used to generate our pseudo-labels. It is the validated state of the art for this exact problem.

#### 4.1 Why transformers belong here

**Semantic argument**: Ink is text. Strokes form letters, letters form words. A faint stroke is more likely real ink if surrounding spatial context is consistent with a Greek letterform. Self-attention captures these long-range dependencies; a CNN with local receptive fields cannot.

**Depth argument**: The Z-stack is a 3D volume. Ink appears as a consistent high-density band across several Z layers — not a single slice. The model should reason jointly over Z depth and spatial position simultaneously.

**The GP-winner argument**: TimeSformer-based attention on CT Z-stacks is not a hypothesis — it produced the first readable Herculaneum transcription. The question is not "will it work?" but "can we implement and tune it effectively with our data?"

---

#### 4.2 Option A — TimeSformer (proven architecture, strongest structural fit)

**What it is**: TimeSformer (*Time-Space Transformer*, Facebook AI Research 2021) designed for video — 3D inputs of T frames × H × W. Core innovation: **divided space-time attention** alternates:

1. **Temporal attention** — each patch attends across all T frames at the same spatial position
2. **Spatial attention** — each patch attends across all N spatial patches at the same frame

Cost: O(N·T²) + O(T·N²) — linear scaling, feasible in practice vs intractable O((TN)²) for full joint attention.

**Mapping to CT Z-stacks**:

| Video (TimeSformer) | CT Z-stack |
|---|---|
| T frames in time | 33 Z layers in depth |
| Spatial H×W per frame | Spatial H×W per Z layer |
| Temporal correlation = motion | Z correlation = depth continuity of ink |
| Task: classify clip | Task: segment ink at each (x,y) |

Z-depth attention: *"is there a consistent ink signal at this (x,y) across all 33 Z layers?"* — the exact physical question of ink detection.

Spatial attention: *"does this location's context match a Greek letterform?"* — the semantic question.

**Architecture for dense prediction**:

```
Input (B, 33, H, W)
    │
    │  patch embed p×p (p=16 recommended)
    ▼
Tokens: (B, 33×N, C)    N = (H/p)×(W/p), C = embed dim (192)
    │
    │  + spatial pos embed + Z-depth pos embed
    ▼
┌─────────────────────────────────────────┐
│  TimeSformer Block × L (L=8)            │
│                                         │
│  1. Z-depth attention (same spatial)    │
│       each (x,y) token attends across  │
│       all 33 Z positions                │
│           ↓                             │
│  2. Spatial attention (same Z layer)    │
│       each token attends across         │
│       all N spatial patches             │
│           ↓                             │
│  3. FFN (MLP, 4× hidden expand)        │
└─────────────────────────────────────────┘
    │
    │  reshape tokens → (B, C, H/p, W/p)
    ▼
Decoder (transposed conv or bilinear upsample ×p)
    │
    ▼
Output (B, 1, H, W)    ink probability logits
```

**Pretrained weights**: TimeSformer-Base pretrained on Kinetics-400. Transfer is imperfect (RGB video ≠ CT grayscale) but spatial attention patterns generalise. Strategy: fine-tune patch embedding from scratch with lower LR, keep transformer blocks at full LR.

**Patch size tradeoff**:
- `p=8` → 33 × 32² = 33,792 tokens — very high memory
- `p=16` → 33 × 16² = 8,448 tokens — recommended starting point

---

#### 4.3 Option B — Hybrid CNN + Swin Transformer bottleneck

Keep the 3D Conv encoder for depth fusion, replace only the bottleneck with Swin. Simpler, lower memory, easier to ablate.

```
3D encoder (Conv3d ×3–4, with skip connections)
    → Z attention pool → (B, C, H', W')
    → flatten: (B, H'×W', C)
    → Swin Transformer (4 layers, window 8×8)
    → reshape: (B, C, H', W')
    → U-Net 2D decoder (with skip connections)
    → head
```

**Tradeoff vs TimeSformer**: Faster and lower memory, but Swin sees only the *collapsed* Z representation — it never directly reasons about Z depth. TimeSformer handles both Z and spatial attention natively.

---

#### 4.4 Comparison

| | TimeSformer | Hybrid CNN-Swin |
|---|---|---|
| Z-depth reasoning | Native (explicit per-token Z attention) | Indirect (CNN pools Z first) |
| Spatial attention | Native | Native (Swin) |
| GP-winner validated | Yes — directly | Partially |
| Pretrained weights | Kinetics-400 (partial transfer) | ImageNet (CNN + Swin both) |
| Memory @ 256px, p=16 | ~8.5k tokens — manageable | Low (pooled map) |
| Implementation complexity | High | Medium |
| Ablation ease | Harder | Easier |

#### 4.5 Recommended experiment sequence

1. Establish `SegmentInkNetV2` CNN baseline (items 1–9)
2. Hybrid CNN-Swin ablation — fast check: does spatial attention help at all?
3. If CNN-Swin beats CNN by > 1% F0.5: proceed to TimeSformer
4. TimeSformer with pseudo-label fine-tuning — this is the target architecture
5. Evaluate all with cross-segment validation (item 2.3)

#### 4.6 Risk table

| Risk | Likelihood | Mitigation |
|---|---|---|
| Overfitting (10 train segs) | Low | Diverse data + weight decay |
| Memory OOM (TimeSformer, p=16, 256px) | Medium | Gradient checkpointing, reduce to p=16 strictly |
| No gain over CNN despite GP validation | Low | GP winner proved it works; more likely implementation issue |
| Pseudo-label noise confuses attention | Low | Ignore band at loss level handles this |
| Pretrain mismatch (video → CT) | Medium | Fine-tune patch embed from scratch; freeze blocks initially |
