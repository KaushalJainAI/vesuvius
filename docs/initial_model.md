# Kaggle Segment Training — Baseline Model Architecture

## Overview

The Kaggle segment training notebook (`notebooks/kaggle_segment_train.ipynb`) trains a compact 3D→2D CNN called **SegmentInkNet** on PHercParis4 labelled scroll segments. The goal is per-pixel ink detection from 33-layer CT Z-stacks.

Total parameter count: **~0.24 M** — deliberately small to fit within Kaggle's GPU memory and 7.5-hour training budget.

This is the **baseline**. For the planned improvements toward TimeSformer, see [plan/model_improvement.md](../plan/model_improvement.md).

---

## Context — What the GP Winner Used

The pseudo-labels this model trains on were generated by the **Grand Prize winning team (Nader / Farritor / Schilliger, 2023) using a TimeSformer-based architecture**. That model applied divided space-time attention natively over CT Z-stacks — reasoning jointly over depth and spatial position — and was the first to produce a readable Herculaneum transcription.

SegmentInkNet is a lightweight starting point. The target end-state architecture is TimeSformer.

## SegmentInkNet Architecture

```
Input  (B, 33, H, W)         ← 33-layer Z-stack patch, float32 in [0, 1]
         │
         │  unsqueeze(1)
         ▼
      (B, 1, 33, H, W)        ← add channel dim for Conv3d

── 3D Encoder ──────────────────────────────────────────────────────────────────

  enc1:  Conv3d(1→16,  k=3, pad=1) → BN3d → ReLU  ×2
  pool1: MaxPool3d(kernel=(2,1,1))              Z: 33 → 16

  enc2:  Conv3d(16→32, k=3, pad=1) → BN3d → ReLU  ×2
  pool2: MaxPool3d(kernel=(2,1,1))              Z: 16 → 8

  enc3:  Conv3d(32→64, k=3, pad=1) → BN3d → ReLU  ×2

  zpool: AdaptiveAvgPool3d((1, None, None))     Z:  8 → 1
         squeeze(2)

── 2D Decoder ──────────────────────────────────────────────────────────────────

  dec1:  Conv2d(64→32, k=3, pad=1) → BN2d → ReLU
  dec2:  Conv2d(32→16, k=3, pad=1) → BN2d → ReLU
  head:  Conv2d(16→1,  k=1)               ← raw logits

Output (B, 1, H, W)          ← ink probability logits (sigmoid at inference)
```

### Key Design Decisions

| Decision | Rationale |
|---|---|
| 3D encoder, 2D decoder | CT data has rich Z-depth structure; spatial (H×W) layout preserved for segmentation |
| `MaxPool3d(2,1,1)` — Z-only pooling | Halves Z resolution while preserving spatial detail |
| `AdaptiveAvgPool3d((1, None, None))` | Collapses any Z depth to 1 — model accepts any Z-depth input |
| Small channel widths (16, 32, 64) | Keeps param count ~0.24 M for Kaggle budget |
| No skip connections | Simplicity for baseline; skip connections are the first planned improvement |

## Building Blocks

### `_conv3d_block(ci, co)`
```
Conv3d(ci→co, k=3, pad=1, bias=False) → BatchNorm3d → ReLU
Conv3d(co→co, k=3, pad=1, bias=False) → BatchNorm3d → ReLU
```

### `_conv2d_block(ci, co)`
```
Conv2d(ci→co, k=3, pad=1, bias=False) → BatchNorm2d → ReLU
```

## Data Pipeline — SegmentStreamDataset

An `IterableDataset` that keeps only one segment's volume in RAM at a time.

**Per-segment flow:**
1. Load 33-layer Z-stack as `uint8` → `float32 / 255.0`
2. Load pseudo-label TIFF → `float32 / 255.0`
3. Derive binary mask from `vol.max(axis=0) > 0`
4. Sample `patches_per_seg` random 256×256 patches with:
   - ≥ 60% of patch within papyrus mask
   - 50% chance to skip near-background patches
5. Augment: horizontal/vertical flip + 0°/90°/180°/270° rotation

**Split (local, all 11 segments):**
- Train: 10 segments
- Val: `20231221180251` (smallest, ~3.5 GB)

---

## Loss Function — `masked_soft_bce_dice`

Combined BCE + soft-Dice with an ignore band on ambiguous pseudo-label regions.

```
supervise = mask & (target < 0.4 OR target > 0.6)

L = 0.5 × BCE(logits[supervise], target[supervise])
  + 0.5 × SoftDice(sigmoid(logits), target, supervise_mask)
```

Pixels with pseudo-label value 0.4–0.6 are excluded — the GP-winner model was uncertain there.

---

## Evaluation Metric — F0.5

```
F0.5 = (1 + 0.25) × precision × recall / (0.25 × precision + recall)
```

Weighted toward **precision**. Threshold: **0.4**.

| Hyperparameter | Value |
|---|---|
| `patch_size` | 256 × 256 |

| `patches_per_seg` (val) | 96 |

| `grad_accum` | 4 (effective batch = 16) |
| `num_epochs` | 60 |
| `lr` | 1e-4 |
| `weight_decay` | 1e-4 |
| `ignore_band` | (0.4, 0.6) |
| `threshold` | 0.4 |
| `max_train_hours` | 7.5 h (Kaggle only) |

### Optimizer & Scheduler

- **AdamW** with `weight_decay=1e-4`
- **CosineAnnealingLR** over 60 epochs
- **Mixed precision (AMP)** via `torch.amp.autocast` when CUDA available
- **Gradient accumulation** over 4 steps

---

## Inference

Sliding-window over the full segment:
- `patch_size = 256`, `stride = 128` (50% overlap)
- Overlapping predictions averaged (`prob / wsum`)
- Output masked by papyrus mask

Outputs: `{seg_id}_prob.npy` (float32), `{seg_id}_prob.tif` (uint8 TIFF).

## Known Weaknesses (Baseline)

| Weakness | Impact | Fix (see improvement plan) |
|---|---|---|
| No skip connections | Decoder loses all spatial detail from encoder | U-Net skip connections (Tier 1) |
| `AdaptiveAvgPool` averages Z | Loses per-layer depth signal | Z attention pool (Tier 2) |
| Tiny capacity (0.24 M params) | Underfits 10 diverse segment textures | 4-stage encoder → 3.5 M (Tier 1) |
| Flip+rotate only augmentation | Limited degradation simulation | Elastic, noise, Z-dropout (Tier 1) |
| BCE+Dice only | Ignores class imbalance | Focal loss (Tier 2) |
| Local Conv only | No long-range spatial/depth reasoning | TimeSformer (Tier 4) |
| Pseudo-label ceiling | Cannot outperform GP-winner labels alone | Iterative refinement (Tier 3) |

## Baseline vs Planned V2 vs Target

| | SegmentInkNet (baseline) | SegmentInkNetV2 (next) | TimeSformer (target) |
|---|---|---|---|
| Params | ~0.24 M | ~3.5 M | ~121 M (Base) |
| Z reasoning | AdaptiveAvgPool | Learned Z attention | Native Z-depth attention per token |
| Spatial reasoning | Local Conv | Local Conv + skip | Global space-time attention |
| Skip connections | No | Yes | N/A (token-based) |
| Pretrained weights | No | No | Kinetics-400 (partial transfer) |
| GP-winner validated | No | No | Yes — same architecture class |

## Segment Track vs Fragment Track

| | SegmentInkNet (segment track) | VesuviusNet (fragment track) |
|---|---|---|
| Input Z-depth | 33 (Z-agnostic via AdaptiveAvgPool) | 65 (fixed) |
| Encoder channels | 16 → 32 → 64 | 32 → 64 → 128 |
| Decoder channels | 64 → 32 → 16 | 128 → 64 → 32 → 16 |
| Parameters | ~0.24 M | ~1 M |
| Patch size | 256 | 224 |
| Training data | 11 labelled segments (pseudo-labels) | Kaggle fragments (manual labels) |
| Loss | BCE + Dice + ignore band | BCE + Dice |
